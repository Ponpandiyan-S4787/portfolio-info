Email
  • The email field must be filled out. It cannot be left blank.
  • The email must match the standard email format (e.g., user{{ at }}example.com).
  • This can be validated using a regular expression (regex) that checks for the presence of "{{ at }}" and a domain suffix.
  • If the form is part of a registration or account creation process, the email must be unique. The system should check the database to ensure that the entered email isn't already associated with an existing account.
  • Some forms may require a specific domain (e.g., for organizational purposes), and the system might check if the domain is valid (e.g., only allowing company emails like user{{ at }}company.com).
